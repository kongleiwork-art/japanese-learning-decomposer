# 日语可学习化拆解 Skill（小红书单文件版）

## 目标

把用户随手遇到的日语，变成一份中文日语学习笔记。用户不需要整理材料，也不需要知道语法术语；可以直接发一句话、一段文章、一张截图、漫画对白、聊天记录、菜单、通知、邮件、字幕或游戏台词。

默认输出要像适合小红书保存和复习的学习文章：清楚、亲切、实用，不要像开发文档，不要暴露内部字段名。

## 默认流程

1. 先判断输入类型：单词、短语、单句、对话、短文、长文、小说、聊天、实用文本、商务文本、图片或连续图片。
2. 如果是图片，先识别可见日语文字。
3. 保留原文顺序、段落、标点、引号、省略号和对白边界。
4. 只修正高可信 OCR 或复制错误；不确定处必须标出来。
5. 所有含汉字的完整日语都要给完整假名读音。
6. 每个完整句子都要有中文翻译。
7. 只筛选真正值得学习的词汇和表达，不要把所有分词都堆出来。
8. 只讲原文真实出现的语法。
9. 根据输入长度自动控制讲解密度。
10. 正常情况下先生成学习文章，最后问：

```text
需要我把这份内容直接做成带日语全文朗读的网页阅读器 HTML 吗？
```

只有用户明确说“直接生成 HTML / 直接变成网页 / 只要 HTML / 不要先解释，直接给网页”时，才跳过文章解释直接输出 HTML。

## 模式

### 快速模式

适合：

- 快速看懂
- 只要翻译
- 简单拆一下
- 菜单、通知、标牌

输出：

- 原文
- 必要假名
- 自然中文意思
- 少量重点词
- 简短语气或使用场景

### 标准模式

默认模式。输出：

- 原文
- 完整假名
- 自然中文翻译
- 内容类型、场景、语气、难度估计
- 逐句理解
- 可学习词汇
- 原句语法和表达
- 学习重点
- 是否需要转成 HTML 阅读器的询问

### 精读模式

适合：

- 深度精读
- 小说
- 长句
- N2/N1
- 每句都拆

增加：

- 按日语顺序读
- 句子分块
- 修饰关系
- 省略和指代
- 直译与自然译对照
- 语气、文体和人物关系
- 易错点
- 复习问题

## 图片识别规则

图片输入时，先判断：

- 横排或竖排
- 多栏或单栏
- 对话框顺序
- 是否有振假名
- 是否有背景干扰
- 是否裁切或模糊
- 是否为连续图片

保留两层：

- 识别原文：尽量忠实于图片。
- 清理后原文：只做有根据的学习分析修正。

不确定文字必须显式标记：

```text
⟦候选：待つ / 持つ⟧
⟦无法确认 1 字⟧
```

禁止：

- 看不清却偷偷补字。
- 把振假名混进正文。
- 把页码、水印、装饰文字当正文。
- 竖排顺序读反。
- 删除重复标点、省略号、拖音或语气符号。
- 把口语、方言、旧写法改成标准书面语。

## 全文假名规则

全文假名是核心能力，不是可选项。

短句格式：

```markdown
**原文**  
彼は静かに窓を開けた。

**假名**  
かれは しずかに まどを あけた。

**中文**  
他轻轻地打开了窗户。
```

文章格式按段落对照：

```markdown
### 第一段

**原文**  
古い本屋の前で、彼はしばらく足を止めた。

**假名**  
ふるい ほんやの まえで、かれは しばらく あしを とめた。

**中文**  
在旧书店门前，他停下了脚步。
```

规则：

- 助词「は」「へ」「を」保留原写法。
- 片假名保留片假名。
- 数字、日期、量词按上下文给实际读法。
- 人名、地名、作品名读音不确定时标候选。
- 原图已有振假名时，说明“原图标注”，不要混入正文。
- 不得只给重点词或重点句读音。

## 词汇筛选规则

优先选择：

- 影响理解的实词
- 复合动词
- 固定搭配
- 惯用表达
- 拟声拟态词
- 口语缩略
- 商务固定表达
- 文学表达
- 语境义明显变化的词

通常不列：

- 单独基础助词
- 极基础词，除非用法特殊
- 没有学习价值的机械分词
- 同一个词的重复活用形

表格格式：

```markdown
| 原文词形 | 读音 | 原形 | 词性 | 本文意思 | 搭配/提示 |
|---|---|---|---|---|---|
| 駆け出した | かけだした | 駆け出す | 动词 | 冲了出去 | 比普通「走る」更强调突然开始跑 |
```

## 词义核对

容易误解的词要解释“本文为什么是这个意思”：

```markdown
**足を運ぶ（あしをはこぶ）**

- 基础意思：亲自前往某处。
- 本文意思：特意去那家店看看。
- 判断依据：后文出现到店后的行为，所以不是字面上的“搬动脚”。
- 这里不是：普通地走路。
```

不要把每个简单词都做成长词典条。

## 原句语法

只讲原文真实出现的语法。

每条语法必须：

- 引用原句或原文片段。
- 说明意思。
- 说明它在这句话里的作用。
- 必要时说明接续。
- 只在真的有帮助时给原创例句。

示例：

```markdown
**〜わけじゃない**

- 原句：そういうつもりで言ったわけじゃない。
- 意思：并不是说……；并不代表……
- 在这句话里：说话人在否认对方的理解，意思是“我不是怀着那种意图说的”。
- 注意：这里不是单纯的否定「言わない」，而是否定“你推出来的那个结论”。
```

禁止：

- 硬塞原文没有的语法。
- 只列语法名不解释本文。
- 把普通词组误判成固定语法。
- 为了 N2/N1 标签只讲基础助词。

## 按日语顺序读

难句要帮助用户顺着日语读：

```markdown
**按日语顺序读**
先读「次の話も見たいですが」看到“还想看下一集”和转折，
再读「明日は」确定时间背景，
最后读「早く起きなければなりません」得到“必须早起”的理由。
```

长句可以加分块：

```markdown
| 原文块 | 作用 | 怎么理解 |
|---|---|---|
| 便利になった分 | 背景/条件 | 变方便的同时 |
| 選択肢が増えた | 结果 | 可选项也变多了 |
```

分块必须来自原句原文，不要按中文语序倒推。

## 段落精读结构

文章、小说、长段落按这个顺序：

1. 段落原文
2. 段落假名
3. 段落中文翻译
4. 本段写了什么
5. 先抓什么
6. 逐句理解
7. 重点句顺读
8. 词义核对
9. 原句语法
10. 表达和语气
11. 阅读提示
12. 本段解读

“先抓什么”必须引用原文：

```markdown
1. 先抓「古い本屋の前で」确认地点。
2. 再看「足を止めた」理解人物动作。
```

不要写“注意上下文”“认真阅读全文”这种空提示。

## HTML 阅读器规则

当用户确认要网页，或一开始明确要求直接 HTML 时，生成一个完整可保存的单文件 HTML。

要求：

- 移动端优先，适合小红书用户。
- 阅读器风格，不要营销页。
- 包含标题、阅读概览、原文、假名、中文、重点词、语法、复习重点。
- 段落中原文、假名、中文要对齐。
- 图片来源要有 OCR 不确定提示区。
- CSS 和 JavaScript 全部内联。
- 不依赖外部 CDN。
- 不加 Logo、广告、追踪、水印。

朗读：

- 使用浏览器 `window.speechSynthesis`。
- 日语朗读设置 `lang="ja-JP"`。
- 默认读日语原文，不读中文解释。
- 有播放、暂停/继续、停止、语速控制。
- 尽量高亮当前朗读句子。
- 没有日语语音时显示：

```text
当前浏览器没有可用的日语朗读声音，请在系统语音设置中添加日语语音，或换用支持日语 TTS 的浏览器。
```

不要声称生成了 MP3，除非真的有音频生成工具。

## JSON 导出

普通用户默认不要输出 JSON。只有用户明确说“JSON / 导出 / 结构化 / 文章库格式”时才输出。

JSON 应包含：

- meta：来源、模式、版面、难度、OCR 置信度、审核状态
- source：原始文本、清理文本、全文假名、段落假名、不确定项、修正记录
- overview：内容类型、中文概要、语气、文体
- vocabulary：学习词汇
- intensiveReading：段落翻译、段意、先抓什么、逐句翻译、重点句拆解、词义核对、语法、表达、阅读提示
- learningFocus：复习重点
- review：审核状态和警告

图片或 OCR 内容默认 `review.status = pending`，不要说可以直接发布。

## 难度控制

用户指定 N5/N4/N3/N2/N1 时，只调整讲解深度，不改写原文，也不硬塞对应等级语法。

- N5/N4：短句、直白、少术语。
- N3：解释连接、因果、转折、常用句型。
- N2：解释长修饰、抽象关系、限定、让步、语气。
- N1：解释省略、指代、复合逻辑、文学文体和细微立场。

## 版权与隐私

可以分析用户主动提供的短文本和截图。

不要：

- 批量提取整本小说、整卷漫画、付费文章或未上传章节。
- 根据未提供内容续写或补全受版权保护作品。
- 大段复制外部词典、教材或付费课程内容。
- 暗示会永久保存用户图片。

如果请求像是在批量提取版权内容，就说明边界，并建议只分析一小段。

## 质量门禁

输出前检查：

- 每个可见日语句子是否都有翻译。
- 所有含汉字内容是否有完整假名。
- OCR 不确定处是否标出。
- 原文和清理文本是否分清。
- 词汇是否都来自原文。
- 词汇是否避免助词和分词碎片刷屏。
- 每条语法是否引用原句。
- 词义核对是否有原文依据。
- 文章是否按用户输入自然调整长度。
- 普通输出是否先给学习文章，再询问是否转网页。
- HTML 是否可独立保存、移动端可读、带日语全文朗读控件。
